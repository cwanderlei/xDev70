1. Configurando os compiladores instalados no seu sistema
 
Um dos passos mais importantes na configuração da xDev Studio é sem dúvida a configuração dos compiladores instalados em seu sistema. Para acessar este tela de configuração, vá no menu FERRAMENTAS > CONFIGURAR COMPILADORES. Uma janela semelhante à figura abaixo será exibida:
   
 

Em destaque na cor vermelha observamos a listagem dos scripts de compilação instalados na sua xDev Studio.
Em destaque na cor azul observamos uma lista dos paths já configurados para o compilador específico selecionado na listagem encontrada na parte superior da janela.
   
 
Na janela de Compiladores você deve selecionar qual script você deseja configurar. Pode-se fazer isto clicando sobre o item desejado no listbox (em destaque na cor vermelha na imagem acima) - isto se faz necessário pois a xDev Studio possui diversos scripts de compilação para os compiladores e ferramentas gráficas mais comuns tais como Clipper, Harbour, xHarbour, FiveWin, MiniGUI, hwGUI, Blinker, MinGW, Borland BCC, Microsoft Visual C entre outros.
   
 
Após selecionar o script desejado clique sobre o botão Adicionar para abrir a tela Configurar Paths. Será exibida a janela onde deve ser colocado os diretórios principais para compilação, observe na figura abaixo os campos principais em destaque:
   
 

 
 
   
 
Nesta tela você deve informar os paths onde estão instalados os arquivos principais para uso nos seus scripts de compilação.

Basicamente deve-se ter em mente o seguinte: a xDev Studio configura as variàveis de ambiente de acordo com os valores preenchidos nesta tela, portanto, é importante que você configure corretamente as opções existentes nesta janela. Veja quais são os principais campos:
   
 
 
  • Versão do Compilador
    • Informe neste campo (destacado em azul na imagem acima) um título que será utilizado para referência futura sobre este perfil de configuração. Utilize algo bem descritivo como "Clipper 5.3 e Blinker 7" ou "xHarbour 0.99.61 e Borland BCC" ou ainda "FiveWin 2.7 June 2006 & BCC 5.5.1".
     
    		  
     
  • SET PATH
    		•
    Você deve especificar neste campo os PATHs onde estão instalados os arquivos principais para uso de seu script. Coloque neste campo separados por ponto-e-virgula os diretórios onde estão os seguintes arquivos:
     
  • A pasta do teu compilador, ou seja, a pasta onde está o arquivo principal do seu compilador (conforme apropriado) que pode ser CLIPPER.EXE ou o HARBOUR.EXE. Geralmente incluimos a pasta C:\CLIP53\BIN ou C:\XHARBOUR\BIN por exemplo, indicando assim onde estará o aplicativo utilizado para o processo de compilação de seus futuros projetos.
     
  • Quando apropriado é necessário informar o caminho completo da pasta BIN do teu linkeditor (Borland BCC / BLINKER / MinGW / Pelles C ou MSVC) exemplo: C:\BCC55\BIN ou C:\MINGW\BIN.
     
  • No caso das LIBs gráficas é necessário informar neste campo a PASTA PRINCIPAL ou PASTA RAIZ onde você instalou os arquivos, como por exemplo: C:\MINIGUI ou C:\FWH.
     
    Para incluir mais de um PATH você pode utilizar o assistente de Diretórios clicando no botão destacado na cor verde na imagem acima. Após configurar as pastas corretas a xDev Studio irá mostrar na parte inferior da janela (destacado em vermelho na imagem acima) a pasta onde está o compilador e a versão do mesmo, bem como a versão do teu linkeditor e ferramenta gráfica (se houver).

    DICA: Para incluir neste campo o seu valor atual de SET PATH conforme configurado em seu AUTOEXEC.BAT ou AUTOEXEC.NT, use a expressão %PATH%; esta mesma dica vale para os campos SET INCLUDE, SET LIB e SET OBJ com as constantes %INCLUDE%, %LIB% e %OBJ% respectivamente.
     
    		•   
     
  • SET INCLUDE
    • Este campo deve conter todos os paths de pesquisa que serão necessários na variàvel de ambiente SET INCLUDE para o processo de compilação de seu aplicativo. Lembre-se que Isto se aplica tanto para o compilador como para o linkeditor; sendo assim normalmente colocamos algo como C:\CLIP53\INCLUDE ou C:\HARBOUR\INCLUDE;C:\BORLAND\BCC55\INCLUDE;C:\FWH\INCLUDE.
     
    		  
     
  • SET LIB
    • Assim como o campo anterior, deve conter os PATHs de pesquisa dos arquivos *.LIB necessários para compilação e linkedição do seu aplicativo. Geralmente indicamos aqui o C:\CLIP53\LIB ou C:\HARBOUR\LIB;C:\BCC55\LIB\PSDK;C:\BCC55\LIB. Quando houver necessidade este campo também deve incluir assim como o campo anterior a pasta LIB de sua ferramenta gráfica.
     
    		 
     
  • SET OBJ
    • Este campo deve conter quando necessário as pastas de pesquisa dos arquivos *.OBJ necessários para compilação e linkedição do seu aplicativo.
     
    		 
     
  • Defines específicas ...
    •  Permite informar diretrivas #DEFINE para uso específico deste compilador.
     
    		  
     
  • Checar sub-pastas ...
    • A opção Checar sub-pastas automaticamente (conforme destacado na imagem acima em amarelo) determina se logo após ao ser preenchido o valor de SET PATH, a xDev Studio deve preencher automaticamente pelos valores de SET INCLUDE, SET LIB e SET OBJ de acordo com a estrutura das pastas informadas em SET PATH.
     
    		  
       
     
    Após configurar corretamente as pastas do seu compilador clique em Ok a janela atual será fechada e será exibida janela anterior de configuração dos compiladores. Você pode ter vários compiladores instalados e de versões diferentes para o mesmo script.

    Observe na figura abaixo que para o mesmo script foram cadastrados 4 (quatro) versões diferentes do mesmo compilador:
       
     

     
     
       
     
    Após configurar adequadamente os paths dos seus compiladores você já poderá vincular nos seus projetos os devidos perfis de configuração desejados.

    2. Criando um novo projeto
     
    Para criar um novo projeto na xDev Studio vá no menu ARQUIVO > NOVO > PROJETO e salve o seu projeto em um local de sua preferência. Após isto utilize o menu PROJETO para adicionar ou remover arquivos em seu projeto aberto atualmente.
      Subir
       
    3. Adicionando ou Removendo arquivos ao seu projeto
     
    Com o devido projeto aberto basta clicar em PROJETO > ADICIONAR ARQUIVOS para incluir outros módulos ou utilize a opção PROJETO > REMOVER para excluir algum módulo já existente de seu projeto.
      Subir
       
    4. Selecionando um compilador para seu projeto
     
    Com o projeto aberto vá no menu PROJETO > PROPRIEDADES e após isto no painel de opções localizado à esquerda da janela, clique em SELECIONAR COMPILADORES.

    Na guia que se surge, selecione um tipo de compilador na primeira lista denominada "Scripts de compiladores instalados" que aparecerá no topo direito da mesma janela. Feito isto, deve-se selecionar na segunda lista logo abaixo, em quais versões dos compiladores instalados você deseja que seu projeto seja compilado. Após selecionar uma versão desejada para compilar seu projeto deve-se marcar qual é será a versão default de compilação; fazemos isto clicando na versão desejada do compilador e após isto selecionando o item destacado em vermelho "Usar este script durante o processo de desenvolvimento"

    Depois disto, você já poderá acessar o item "Configurar compiladores selecionados" do lado esquerdo do painel para configurar os detalhes de cada compilador.
      Subir
       
    5. Importando um projeto já existente para a xDev Studio
     
    Se estiver desejando importar um projeto de uma versão antiga da própria xDev (tal como a 0.33 ou 0.34) basta apenas ir em ARQUIVO > ABRIR e selecionar o arquivo de projeto criado com sua versão antiga. Após isto basta ler com atenção as orientações que a xDev Studio irá lhe exibir na tela e pronto!
       
      Para importar a listagem de arquivos de um projeto já existente, mas criado com outra ferramenta, execute os seguintes passos:
       
     
    Com um projeto aberto clique em ARQUIVO > IMPORTAR e selecione o formato do arquivo que deseja importar. Após fazer isto será exibida a janela "Importar projeto do..." onde você deverá selecionar o projeto que contém os nomes dos arquivos para importação. Após selecionar o arquivo original a xDev Studio importar a lista com os nomes dos arquivos de seu projeto antigo e irá adicionar ao seu projeto atualmente aberto.
